2️⃣ What It Measures Mathematically

Now the important part.

Your script enforces structured metrics.

🔹 A) Attitude Stability (Primary Objective)
Formula used:
angle_error = |roll| + |pitch|

Then averaged over entire mission:

mean(angle_error)

This measures:

How much the drone tilts while moving.

Lower = better stabilization.

This directly checks what you trained for.

🔹 B) Angular Velocity Magnitude

Computed as:

ang_mag = sqrt(wx² + wy² + wz²)

This measures:

How violently the drone rotates.

High angular velocity = oscillation
Low angular velocity = smooth control

This checks whether the PPO policy is stable or twitchy.

🔹 C) Recovery Time

When instability occurs:

If:

angle_error > threshold

Then recovery time is measured until:

angle_error < stable_threshold

Mathematically:

Recovery time = number of steps until re-stabilization

This measures disturbance rejection speed.

This is critical.

A good stabilizer:
• May tilt briefly
• But recovers fast

🔹 D) Consistency (Variance)

Computed as:

std(angle_errors)

Standard deviation of tilt.

Low variance = confident controller
High variance = nervous / oscillatory controller

This checks policy smoothness across time.

🔹 E) Mission Completion

Binary:

completed = True if all checkpoints reached

This checks whether instability accumulates over long flight.

If stabilization is poor,
drift accumulates,
mission fails before final checkpoint.
