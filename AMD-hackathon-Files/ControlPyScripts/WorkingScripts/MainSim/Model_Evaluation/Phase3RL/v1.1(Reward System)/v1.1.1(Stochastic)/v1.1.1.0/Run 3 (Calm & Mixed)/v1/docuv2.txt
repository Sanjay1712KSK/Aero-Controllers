3️⃣ What Happens Inside Each Episode
Step-by-step loop:
1️⃣ Model predicts action
action, _ = model.predict(obs, deterministic=True)

This means:

No randomness

Pure learned behavior

Real policy evaluation

2️⃣ Wind is injected

Wind is generated based on regime:

Calm → small disturbance

Strong → large disturbance

Mixed → moderate disturbance

This simulates environmental anomalies.

3️⃣ Drone moves

Velocity command:

vx = action + wind

So the model must:

Counteract wind

Stabilize itself

4️⃣ Sensor logging

Every step logs:

roll

pitch

yaw

linear velocity

angular velocity

wind vector

This gives you full telemetry trace.

You can reconstruct flight entirely from CSV.

4️⃣ What the Script Measures

It calculates 3 core metrics:

A) Angle error
abs(roll) + abs(pitch)

Measures:
How tilted the drone is.

Lower = better stability.

B) Angular velocity magnitude
norm(ang_vel)

Measures:
How violently it is rotating.

Lower = smoother control.

C) Drift
norm(linear_velocity)

Measures:
How much it is drifting or oscillating.

Lower = tighter control.

5️⃣ The Stability Score (/100)

Each episode gets:

25 points for angle stability

25 for angular smoothness

25 for drift control

25 for surviving full 200 steps

Total = 100 max

This gives you a clear quantitative performance measure.

7️⃣ Bad Behavior Detection

Trigger condition:

abs(roll) > 0.45 OR abs(pitch) > 0.45

Meaning:
Drone is dangerously tilted.

When triggered:

Save snapshot

Record 5-second highlight video

Saved to:

snapshots/bad/
highlight_videos/bad/

This documents failure events.

8️⃣ Good Recovery Detection

Trigger condition:

Wind strong AND
roll & pitch very small for 25 consecutive steps

Meaning:
Drone is stable despite disturbance.

When triggered:

Save snapshot

Record 5-second highlight

Saved to:

snapshots/good/
highlight_videos/good/

This documents robustness.
